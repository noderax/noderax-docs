# Local Development And Release Flow

## Development

The main repos are developed independently, then integrated through release
metadata.

<Steps>
  <Step>
    API, web, agent, platform-release, marketing, and docs repos evolve
    independently in local development.
  </Step>
  <Step>
    Shared product behavior is reconciled through explicit refs, generated
    artifacts, and release metadata rather than a single monorepo package graph.
  </Step>
  <Step>
    Release repos then consume those outputs into versioned artifacts and CDN
    manifests.
  </Step>
</Steps>

## OpenAPI docs flow

- `noderax-api` exports a deterministic schema snapshot
- `noderax-docs` syncs that snapshot locally
- Fumadocs renders generated endpoint pages from the checked-in schema

<CodeBlock>
```bash
cd noderax-api
pnpm openapi:export

cd ../noderax-docs
pnpm sync:openapi
```
</CodeBlock>

## Control-plane release flow

- `noderax-platform-release` reads `apiRef` and `webRef`
- it builds the platform bundle
- it publishes bundle artifacts and manifests to the CDN
- installer-managed deployments pull from the CDN latest manifest
